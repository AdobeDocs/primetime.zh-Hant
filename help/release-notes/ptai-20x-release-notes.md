# PrimetimeAd Insertion20.12.1發行說明

PrimetimeAd Insertion20.12.1發行說明說明2020年PrimetimeAd Insertion的新增或變更專案、已解決的問題及已知問題。

**變更**

### 20.9.3版

**時間：** 東部時間2020年9月30日星期三凌晨3:30至早上6:30

**變更**

* 新增BootstrapAPI引數 `ptparallelstream`. 這可讓客戶讓播放器要求同時使用CMAF未嵌入音訊或視訊資料流，以確保音訊和視訊曲目中的廣告一致。 將引數值設為true可啟用此功能，或省略以停用。

### 20.9.2版

**時間：** 2020年9月15日星期二美國東部時間凌晨3:30至早上6:30

**增強功能**

* 支援使用納入非線性廣告型別 `EXT-X-MARKER` 標籤之間。
如需詳細資訊或要啟用此功能，請聯絡您的技術支援代表。

* 如果提供者花太長的時間回應，提供可限制整體廣告解析時間的支援。 若要啟用限制，請設定啟動程式API引數 `ptadtimeout` 變更為以毫秒為單位的值。

   >[!NOTE]
   >
   >此逾時僅適用於廣告請求，不適用於廣告創意請求。

### 20.9.1版

**時間：** 2020年9月1日（星期二），東部時間凌晨3:30至上午7:30

**變更**

* 修正使用HLS/CMAF的客戶，其中EXT-X-MAP有時遺失CDN權杖或EXT-X-MAP標籤有時從DVR視窗中錯誤轉出的問題。

### 20.7.1版

**時間：** 2020年7月9日（星期四），東部時間凌晨03:00至凌晨05:00

**新功能和增強功能**

* SCTE35增強功能，可使用「提供者廣告開始/結束」訊息或「中斷開始/結束」訊息來識別提示。

* 更新X-ADOBE-AI-X1標頭，並提供協助疑難排解的額外資訊。

* 增強量度彙總。

* 增強工作階段統計面板的SSAI主控台控制面板

### 20.6.2版

**時間：** 2020年6月18日（星期四），東部時間凌晨03:00至凌晨04:00

**增強功能**

改善需要毫秒精確度的視訊使用者端的資料流同步處理。 請聯絡Adobe支援，為以下專案啟用毫秒精確度： `#EXT-X-PROGRAM-DATE-TIME tags`.

### 20.6.1版

**時間：** 2020年6月2日（星期二），東部時間凌晨03:00至凌晨05:00

**新功能**

請聯絡Adobe支援，透過伺服器端設定來啟用下列新功能：

* 資訊清單操作：現在可以在HTTP和HTTPS之間轉換HLS區段和資源URL，以減少後端請求上的TLS交握次數來提高效能。 它也可用來將廣告/內容片段整合到相同的CDN上。

* 長格式VOD：改善API，使用長格式VOD資產維持工作階段持續運作。

**錯誤修正**

* 修正無論要求的原始通訊協定為何，一律根據http通訊協定要求WebVTT片段的問題。

* 修正從廣告切換回內容時，從播放清單頂端移除EXT-X-DISCONTINUITY標籤的問題。 請聯絡Adobe支援以啟用此修正。

### 20.5.1版

**時間：** 2020年5月5日星期二美國東部時間凌晨04:00至凌晨05:00

* 已修正問題，以確保在傳送If-Modified-Since標頭時提供正確的CORS標頭。

### 20.3.4版

**時間：** 2020年4月1日（星期三），東部時間凌晨03:00至凌晨04:00

* 修正在VOD/WebVTT中插入廣告後，字幕不同步的問題。

* 安全性更新。

### 20.3.3版

**時間：** 2020年3月26日（星期四），東部時間凌晨03:00至凌晨04:00

* SSAI 4XX和5XX回應現在可以正確提供CORS相關的標頭，讓跨網域javascript webview使用者端可以成功讀取錯誤回應。

* 修正X-Forwarded-For標頭的問題，此問題發生在將IPv6位址傳遞至廣告伺服器時，其URL編碼不正確。

* 修正CMAF/未內嵌音訊串流的問題，在某些情況下，EXT-X-MEDIA-SEQUENCE編號會不正確地增加。

### 20.3.2版

**時間：** 2020年3月11日（星期三），東部時間上午5:30至上午7:00

* 改善SCTE35訊號處理。

* 維護更新。

### 20.3.1版

**時間：** 2020年3月5日（星期四），東部時間凌晨02:30至凌晨04:30

* 效能改善：

   * 新增對master/media m3u8資料清單的快取支援。 這些資訊清單現在會回應Cache-Control： public和Max-Age標頭，這通常可以改善視訊開始效能。

   * 新增對強制透過http擷取https創作的支援，這也可以改善視訊開始效能。

* 安全性與維護修正。

### 20.2.1版

**時間：** 2020年2月13日星期四美國東部時間凌晨04:30至凌晨05:30

* 新增對拼接廣告資產的支援，這些資產包含根據語言/轉碼器/位元速率的多個純音訊資料流。
* 微幅效能增強和維護更新。

### 20.1.3版

**時間：** 2020年1月28日星期二美國東部時間凌晨2:00至凌晨03:00

* **VMAP搭配FER支援nbc CueFormat**

   將FER資料流的提示轉換為FW時間表覆寫引數，當 `ptcueformat=nbc` 已使用，且串流是包含資訊清單內提示和已製作廣告的VOD串流。

* 在轉送給協力廠商廣告提供者/CDN之前，先處理HTTP標題中的使用者代理欄位。

* 在傳送至Auditude和其他廣告提供者、CDN之前，從使用者代理HTTP標題中篩選掉控制/不可列印的字元（ASCII代碼&lt; 32）。 用於此類無效標頭的稽核廣告呼叫過去會失敗。

* 從NetStorage群組清除舊的V1物件，使物件計數在Akamai的安全限制內。
